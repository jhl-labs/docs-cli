docs-cli

docs-cli init . 한 번이면 어떤 저장소든 하나의 표준 문서 스키마로 docs/ 폴더가 생깁니다. AI 에이전트로 코드를 리버스 엔지니어링해 채우고, 정합성을 기계가 판정하며, HTML·XML로 렌더링해 웹 서비스에 그대로 이식합니다.

왜 docs-cli 인가

표준 스키마

섹션·문서·챕터·프론트매터가 코드로 고정됩니다. 프로젝트가 달라도 산출물 형태가 같습니다.

리버스 엔지니어링

generate --agent claude 로 코드를 읽어 문서를 채웁니다. claude·codex·gemini·opencode 지원.

검증 게이트

validate 가 필수 문서·프론트매터·챕터 누락을 기계적으로 판정합니다. 눈대중이 아닙니다.

웹 이식

고정 프론트매터 + 구조적 XML 로 렌더링해 웹 서비스가 매핑 없이 ingest 합니다.

다운로드 · 설치

# 최신 공개 바이너리 설치
curl -fsSL https://jhl-labs.github.io/docs-cli/install.sh | sudo bash

# 버전 고정 / 사용자 경로 설치
curl -fsSL https://jhl-labs.github.io/docs-cli/install.sh | sudo env VERSION=v0.1.0 bash
curl -fsSL https://jhl-labs.github.io/docs-cli/install.sh | INSTALL_DIR="$HOME/.local/bin" bash

Linux · macOS · Windows (amd64 / arm64). 설치 스크립트는 jhl-labs/dist 의 최신 릴리스를 받아 SHA256 으로 검증합니다.

빠른 시작

docs-cli init .                              # 표준 docs/ 트리 스캐폴딩
docs-cli generate . --agent claude           # 에이전트로 문서 채우기
docs-cli validate . --strict                 # 표준 정합성 게이트
docs-cli render . --format html --format xml # 웹 이식용 산출물

CI 에서는 jhl-labs/docs-cli-action 으로 동일 흐름을 자동화합니다.
